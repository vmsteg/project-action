## 🔍 О проекте
 
**TaskFlow** был создан для тех, кто проводит много времени в консоли и хочет эффективно управлять своими задачами. Программа хранит все данные в локальном файле `tasks.json`, поддерживает теги, приоритеты и статусы выполнения.
 
> *"Дисциплина — это мост между целями и достижениями."*  
> — Эта цитата вдохновляет нас делать TaskFlow инструментом, который помогает не просто записывать задачи, а **выполнять** их.
 
Ключевая особенность — минималистичный интерфейс и поддержка **горячих клавиш** <kbd>Ctrl</kbd> + <kbd>N</kbd> для быстрого добавления задачи<sup>[1](#footnote1)</sup>. Вся работа с проектом описана в [официальной документации](https://example.com/docs) (внешняя ссылка).

## ✨ Возможности
 
- 📝 **Создание задач** с заголовком, описанием и дедлайном.
- 🏷️ **Категоризация** с помощью тегов (например, `#работа`, `#личное`).
- 🔔 **Умные напоминания** (пока только в консоли при запуске).
- 📊 **Фильтрация и сортировка**:
  - По статусу (`Выполнено / В процессе`)
  - По приоритету (`Высокий / Средний / Низкий`)
  - По дате
- 💾 **Автосохранение** при каждом изменении.
 
### Структура данных (вложенный список)
- `data/` – папка с пользовательскими данными
  - `tasks.json` – основной файл задач
  - `backup/` – резервные копии
    - `tasks_old.json` – последний бэкап

Ответы на контрольные вопросы
1. Чем отличается вложенная цитата от обычной?
Обычная цитата начинается с одного символа > и представляет собой один уровень вложенности.
Вложенная цитата содержит два (или более) символа >> и используется для цитирования внутри цитаты.
 
Пример обычной цитаты:
 
Это простая цитата.
 
Пример вложенной цитаты:
 
Это первый уровень цитирования.
 
Это второй уровень (вложенная цитата).
Возврат на первый уровень.
 
2. Как сделать сноску в Markdown и где она отображается?
Создание сноски:
 
В тексте ставится метка: [^1] или [^имя]
 
Внизу документа даётся определение: [^1]: Текст сноски
 
Где отображается: Сноски отображаются в самом низу страницы (при рендеринге на GitHub, в VS Code и других платформах).
 
Пример:
 
markdown
Текст со сноской[^1].
 
[^1]: Пояснение к сноске.
3. В чём разница между - [ ] и - [x] в списках задач?
Синтаксис   Значение    Статус
- [ ]   Невыполненная задача    Открыта
- [x]   Выполненная задача  Завершена
Пример:
 
markdown
- [x] Написать документацию
- [ ] Провести ревью кода
4. Какие два способа вставки эмодзи вы знаете?
Прямая вставка символа эмодзи (если клавиатура поддерживает ввод эмодзи):
Пример: ❤️, 🔥, ✅
 
Использование кода эмодзи (текстовый псевдоним, работает на GitHub):
Пример: :heart: → ❤️, :rocket: → 🚀, :white_check_mark: → ✅
 
5. Зачем нужны HTML-вставки? Приведите пример.
Зачем: HTML-вставки позволяют добавить возможности, которых нет в стандартном Markdown: выпадающие списки, видео, аудио, изменение выравнивания, обозначение клавиш.
 
Пример выпадающего списка:
 
html
<details>
  <summary>Нажмите, чтобы увидеть подробности</summary>
  Это скрытый текст, который появляется по клику.
</details>
Пример обозначения клавиш:
 
html
<kbd>Ctrl</kbd> + <kbd>C</kbd>
6. Почему важно использовать правильные имена для внутренних ссылок (якорей)?
Потому что:
 
Неправильный якорь приводит к неработающей ссылке
 
Ссылки на заголовки автоматически генерируются парсером (строчные буквы, пробелы заменяются дефисами)
 
Правильные якоря обеспечивают удобную навигацию и профессиональный вид документа
 
7. Что произойдёт, если в таблице окажется разное количество столбцов в разных строках?
Markdown-парсер обработает таблицу некорректно:
 
Лишние ячейки могут быть проигнорированы
 
Недостающие ячейки отобразятся как пустые
 
В некоторых парсерах таблица может сломаться (не отобразиться вовсе)
 
Правило: Все строки таблицы должны иметь одинаковое количество ячеек.
 
Пример ошибки (3 столбца в заголовке, 2 ячейки в строке):
 
markdown
| A | B | C |
|---|---|
| 1 | 2 |
→ Строка сломана: ячейка C во второй строке отсутствует.
